# Qraft

**AI-Powered SQL Script Generator** — Transform natural language into production-ready SQL, instantly.

Qraft helps engineering teams write SQL queries faster by leveraging AI with full awareness of your database schema. Upload your table designs, describe what you need, and get SQL that's ready to run.

---

## Features

- **AI-Driven SQL Generation** — Describe your query in plain language (English or Bahasa Indonesia) and get accurate, executable SQL.
- **Schema-Aware** — Upload your database design (DDL, JSON, CSV) so the AI knows your exact tables, columns, and relationships.
- **Production-Ready Output** — Generated SQL uses correct table/column names from your schema — no hallucinated fields.
- **One-Click Copy** — Copy generated SQL to clipboard instantly.
- **Dark-First UI** — Built with shadcn/ui components, designed for developer workflows.
- **API-First Architecture** — All data access goes through API routes. No direct database queries from components.
- **Safe by Design** — Qraft generates SQL but never executes it. Your data stays safe.

## How It Works

```
┌─────────────────┐     ┌──────────────────────┐     ┌─────────────────┐
│  Upload Schema  │────>│  System Prompt + AI  │────>│  SQL Output     │
│  (DDL / JSON)   │     │  Knowledge Base      │     │  (Ready to Run) │
└─────────────────┘     └──────────────────────┘     └─────────────────┘
                              ^
                              │
                     ┌────────────────┐
                     │  User Request  │
                     │  (Natural Lang)│
                     └────────────────┘
```

1. **Upload** your database schema (CREATE TABLE statements, JSON definitions, etc.)
2. **Ask** for what you need in natural language
3. **Get** production-ready SQL tailored to your exact database structure

## Architecture

### API-First Pattern

All data flows through API routes — components never query the database directly.

```
Client/Server Component
  → fetch("/api/...")
    → API Route Handler (src/app/api/)
      → Drizzle ORM (lib/db/)
        → Supabase PostgreSQL
```

## Getting Started

### Prerequisites

- **Node.js** 18.17 or later
- **npm** (or pnpm/yarn/bun)
- **Supabase** project (for PostgreSQL database)

### Installation

```bash
# Clone the repository
git clone https://github.com/your-org/qraft.git
cd qraft

# Install dependencies
npm install
```

### Environment Setup

Create a `.env.local` file in the project root:

```env
# AI Service
AI_API_KEY=your-api-key-here
AI_MODEL=gpt-4o
AI_BASE_URL=                     # optional: custom API endpoint

# Database (Supabase)
DATABASE_URL=postgresql://...     # Supabase connection pooler URL

# App
NEXT_PUBLIC_APP_NAME=Qraft
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### Database Setup

```bash
# Generate migrations from schema definitions
npx drizzle-kit generate

# Apply migrations to database
npx drizzle-kit migrate
```

### Run Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Usage Examples

### Simple Query

**Request:**
> "Tampilkan semua user yang statusnya active"

**Generated SQL:**
```sql
SELECT *
FROM users
WHERE status = 'active';
```

### Aggregation Query

**Request:**
> "Buat report total penjualan per bulan di tahun 2025"

**Generated SQL:**
```sql
SELECT
    DATE_FORMAT(order_date, '%Y-%m') AS month,
    COUNT(*)                         AS total_orders,
    SUM(total_amount)                AS total_revenue
FROM orders
WHERE YEAR(order_date) = 2025
GROUP BY DATE_FORMAT(order_date, '%Y-%m')
ORDER BY month ASC;
```

### Complex Join

**Request:**
> "Get all customers with their latest order and total spend"

**Generated SQL:**
```sql
SELECT
    c.id            AS customer_id,
    c.name          AS customer_name,
    c.email,
    lo.order_date   AS latest_order_date,
    lo.total_amount AS latest_order_amount,
    agg.total_spent
FROM customers c
LEFT JOIN (
    SELECT customer_id, MAX(order_date) AS max_date
    FROM orders
    GROUP BY customer_id
) latest ON latest.customer_id = c.id
LEFT JOIN orders lo
    ON lo.customer_id = latest.customer_id
   AND lo.order_date  = latest.max_date
LEFT JOIN (
    SELECT customer_id, SUM(total_amount) AS total_spent
    FROM orders
    GROUP BY customer_id
) agg ON agg.customer_id = c.id
ORDER BY agg.total_spent DESC;
```
